# [Project Name]: 사용자 친화적 정책 가이드 AI

## 📖 프로젝트 소개 (Introduction)

**폴리스탭**은 복잡하고 파편화된 정책 정보를 사용자가 쉽고 직관적으로 이해할 수 있도록 돕는 AI 기반 정책 가이드 서비스입니다.

수많은 정책이 존재하지만, 정작 필요한 사람에게 닿지 못하는 정보의 비대칭 문제를 해결하기 위해 시작되었습니다. 단순한 텍스트 검색을 넘어, 사용자의 상황에 맞는 맞춤형 정보를 시각적으로 아름답고 편리한 UI를 통해 제공합니다.

---

## 🎯 프로젝트 목표 (Project Goals)

이 프로젝트의 핵심 목표는 **"정책 정보의 접근성 혁신"**입니다.

1.  **정보의 정확성:** 할루시네이션을 최소화하고, 공신력 있는 정부 데이터를 기반으로 정확한 답변을 제공합니다.
2.  **사용자 경험(UX) 극대화:** 딱딱한 공문서 스타일에서 벗어나, 누구나 머무르고 싶은 심미적이고 직관적인 인터페이스를 구축합니다.
3.  **맞춤형 큐레이션:** 사용자의 조건(나이, 소득, 거주지 등)을 분석하여 가장 혜택이 큰 정책을 우선적으로 제안합니다.

---

## 💡 기존 서비스와의 차이점 (Key Differentiators)

Perplexity나 Tavily와 같은 범용 검색 엔진/에이전트와 비교했을 때, **폴리스탭**은 **'정책'이라는 도메인의 특수성**과 **'사용자 경험'**에 집중하여 다음과 같은 차별점을 가집니다.

### 1. Perplexity / Tavily vs. [Project Name]

| 구분 | Perplexity / Tavily | [Project Name] |
| :--- | :--- | :--- |
| **정보의 성격** | **실시간성 (News/Web)**<br>시시각각 변하는 뉴스나 웹 정보 검색에 최적화됨 | **불변성/지속성 (Policy)**<br>정책은 매일 변하지 않음. 정확한 원문(공고문) 해석과 깊이가 중요 |
| **검색 범위** | 전체 웹 (Broad Search) | 신뢰할 수 있는 정부/지자체 데이터베이스 (Domain Specific) |
| **UI / UX** | 텍스트 중심의 챗봇 인터페이스<br>(정보 전달에 치중) | **시각적 몰입감 중심 (Rich UI)**<br>카드 뉴스, 시각화된 대시보드 등 심미적으로 완성도 높은 디자인 제공 |
| **목적** | "무엇이든 물어보세요" | "나에게 맞는 혜택을 찾아주세요" |

### 2. 왜 다른가? (Why It Matters)

* **정책 데이터의 정적 특성 활용 (Static Data Optimization):**
    * 정책은 뉴스처럼 분 단위로 변하지 않습니다. 따라서 실시간 웹 검색보다는 **사전에 구축된 고품질 벡터 데이터베이스(Vector DB)**를 통해 공고문(HWP, PDF)을 정확하게 파싱하고 이해하는 것이 훨씬 중요합니다. 우리는 속도보다는 **'해석의 정확도'**에 집중합니다.

* **심미적인 UI/UX (Aesthetic UI/UX):**
    * 기존의 정부 사이트나 일반 챗봇은 딱딱하고 건조합니다. 우리는 **"보기 좋은 떡이 먹기도 좋다"**는 철학 아래, 사용자가 정보를 탐색하는 과정 자체가 즐거운 경험이 되도록 디자인에 중점을 두었습니다.
